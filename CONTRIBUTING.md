# CONTRIBUTING.md

Как вносить изменения в Enoch-Wiki.

## Для людей

Если вы хотите добавить материал:

1. Сначала найдите или добавьте источник в `SOURCES/`.
2. Затем создайте или обновите `summary` в `NOTES/summaries/`, если источник действительно важен.
3. Если есть отдельная идея, вынесите её в короткую заметку.
4. Свяжите новую заметку с уже существующими.
5. Не удаляйте полезный старый материал без причины.
6. Если есть альтернативная трактовка, лучше добавить новую заметку, чем спорно переписывать старую.

## Для заметок

### Summary
Summary - это не один абзац и не формальная аннотация.  
Это отчёт о прочтении всего источника:
- общий контекст
- структура
- содержание
- мотивы
- связи с базой

Summary всегда хранится в `NOTES/summaries/`.

### Короткая заметка
Короткая заметка нормальна, если она атомарная:
- одна мысль
- один мотив
- одна фигура
- один вывод

### Связи
У каждой новой заметки должна быть хотя бы одна связь с уже существующей.

Если связь естественно есть, можно и нужно добавлять больше.

### Как связывать заметки в Obsidian

Внутри заметок используй Obsidian-wikilinks:

```md
[[summary-1-enoh]]
[[entity-azazel|Азазел]]
[[analysis-potop]]
```

Правила:
- `summary` связывай с source и ключевыми связанными заметками
- короткую заметку связывай минимум с одним уже существующим узлом
- если связь очевидно не одна, добавляй несколько по смыслу
- не ставь ссылки просто ради количества
- не добавляй `.md` внутри wikilinks
- если имя уже есть в базе, лучше использовать его или redirect note, чем заводить дубликат

## Для именования

- Файлы пишутся латиницей, без пробелов
- Wikilinks пишутся без `.md`
- Если имя меняется, старую ссылку лучше сохранить через redirect note

## Для pull request

Если вы хотите предложить правки через GitHub:
- кратко опишите цель
- перечислите основные файлы
- скажите, не ломали ли вы связи
- если это структурная правка, проверьте `NOTES/index.md` и `index.md`

## Хороший формат изменений

Хорошие изменения в этой базе:
- добавляют источник
- улучшают summary
- усиливают связи
- сохраняют граф
- не смешивают несколько идей в одной заметке
